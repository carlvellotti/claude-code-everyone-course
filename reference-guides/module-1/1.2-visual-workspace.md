# Рабочее пространство OpenCode

## Интерфейс терминала

OpenCode работает прямо в вашем терминале, интегрируясь в привычный рабочий процесс.

```
╭──────────────────────────────────────────────────────────╮
│  OpenCode                              MiMo-V2-Pro  🔨   │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  > Ваш вопрос или задача...                              │
│                                                          │
├──────────────────────────────────────────────────────────┤
│  ⏎ отправить  Tab переключить  Ctrl+C выйти  / помошь   │
╰──────────────────────────────────────────────────────────╯
```

## Элементы интерфейса

### Заголовок
- Название: OpenCode
- Текущая модель (например, MiMo-V2-Pro)
- Текущий режим (🔨 build или 📋 plan)

### Область ввода
- Строка для ввода запросов
- Поддержка многострочного ввода (Shift+Enter)

### Статусная строка
- ⏎ — отправить сообщение
- Tab — переключить режим агента
- Ctrl+C — выйти
- / — показать команды

## Режимы агентов

### Build режим (🔨)
Используется для генерации и изменения кода:
- Создание новых файлов
- Редактирование кода
- Запуск команд
- Работа с Git

### Plan режим (📋)
Используется для анализа и планирования:
- Анализ кодовой базы
- Планирование архитектуры
- Объяснение кода
- Рекомендации

### Переключение режимов
Нажмите **Tab** для переключения между режимами. OpenCode запомнит ваш выбор.

## LSP интеграция

OpenCode имеет встроенную поддержку Language Server Protocol:

### Автодополнение
OpenCode понимает контекст вашего кода и предлагает релевантные подсказки.

### Анализ типов
```
> Проверь типы в этом файле и найди ошибки
```

### Рефакторинг
```
> Переименуй функцию processPayment в handlePayment везде в проекте
```

## Советы по эффективности

1. **Используйте Tab** — быстрое переключение режимов экономит время
2. **Задавайте контекст** — чем детальнее запрос, тем лучше результат
3. **Используйте /** — для просмотра всех доступных команд
4. **Читайте ответы** — OpenCode объясняет свои решения
5. **Экспериментируйте** — пробуйте разные формулировки запросов
