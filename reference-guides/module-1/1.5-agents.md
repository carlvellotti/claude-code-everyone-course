# Агенты OpenCode

## Что такое агенты?

Агенты — это специализированные режимы работы OpenCode, оптимизированные для разных задач. OpenCode предоставляет два встроенных агента с возможностью быстрого переключения.

## Встроенные агенты

### 🔨 Build агент

**Назначение**: Генерация и изменение кода

**Когда использовать**:
- Создание новых файлов и функций
- Редактирование существующего кода
- Запуск команд и скриптов
- Работа с Git
- Рефакторинг

**Примеры запросов**:
```
> Создай новый React компонент для карточки товара
> Добавь валидацию email в форму регистрации
> Исправь баг в функции calculateTotal
> Создай API endpoint для пользователей
```

### 📋 Plan агент

**Назначение**: Анализ и планирование

**Когда использовать**:
- Анализ кодовой базы
- Планирование архитектуры
- Объяснение кода
- Рекомендации по улучшению
- Code review

**Примеры запросов**:
```
> Проанализируй структуру проекта и предложи улучшения
> Объясни как работает система аутентификации
> Оцени сложность реализации новой функции
> Составь план рефакторинга модуля оплаты
```

## Переключение агентов

### Через Tab

Нажмите **Tab** для быстрого переключения между режимами.

### Через команды

```
# Переключиться на build
> /build

# Переключиться на plan
> /plan

# Показать текущий режим
> /agent
```

### Автоматическое переключение

OpenCode может автоматически определить нужный режим:

```
> Мне нужно создать новый компонент    # → build
> Мне нужно понять как работает код    # → plan
```

## Сравнение агентов

| Функция | Build | Plan |
|---------|-------|------|
| Создание кода | ✅ | ❌ |
| Изменение кода | ✅ | ❌ |
| Запуск команд | ✅ | ❌ |
| Анализ кода | ✅ | ✅ |
| Объяснение | ✅ | ✅ |
| Планирование | ⚠️ | ✅ |
| Рекомендации | ⚠️ | ✅ |
| Code review | ⚠️ | ✅ |

✅ — оптимально, ⚠️ — возможно, ❌ — не рекомендуется

## Практические примеры

### Пример 1: Использование Build для создания компонента

```bash
# Переключиться на build (Tab)
> Создай компонент UserCard с пропсами:
> - name: string
> - email: string
> - avatar: string
> - onEdit: () => void
>
> Используй styled-components для стилизации.
```

### Пример 2: Использование Plan для анализа

```bash
# Переключиться на plan (Tab)
> Проанализируй структуру проекта.
> Предложи 3 способа улучшения организации кода.
> Оцени сложность каждого изменения.
```

### Пример 3: Комбинированное использование

```bash
# 1. Plan: Анализ
> /plan
> Проанализируй текущую систему аутентификации
> и предложи план улучшений.

# 2. Build: Реализация
> /build
> Реализуй первую фазу улучшений:
> добавь refresh token механизм.

# 3. Plan: Проверка
> /plan
> Проверь реализацию refresh token.
> Есть ли проблемы с безопасностью?
```

## Настройка поведения агентов

### В AGENTS.md

```markdown
# Настройки агентов

## Build агент
- Всегда создавай тесты для новых функций
- Используй TypeScript строгую типизацию
- Следуй ESLint правилам проекта

## Plan агент
- Анализируй с учётом архитектурных принципов
- Учитывай масштабируемость
- Предлагай конкретные решения
```

### В ~/.opencode.json

```json
{
  "agents": {
    "build": {
      "createTests": true,
      "strictTypes": true,
      "lintOnSave": true
    },
    "plan": {
      "detailLevel": "high",
      "includeMetrics": true
    }
  }
}
```

## Пользовательские агенты

### Создание кастомного агента

Создайте файл `.opencode/agents/reviewer.md`:

```markdown
# Агент: Code Reviewer

Специализированный агент для ревью кода.

## Фокус:
- Безопасность
- Производительность
- Читаемость
- Тестируемость

## Формат ответа:
1. Общая оценка (1-10)
2. Критические проблемы
3. Рекомендации
4. Примеры улучшений

## Стиль:
- Конструктивный
- Конкретный
- С примерами кода
```

### Использование

```
> /agent reviewer
> Проведи ревью файла src/api/users.ts
```

## Советы по работе с агентами

1. **Начинайте с Plan** — для сложных задач сначала спланируйте
2. **Переключайтесь через Tab** — это самый быстрый способ
3. **Используйте правильный агент** — build для кода, plan для анализа
4. **Комбинируйте** — plan → build → review
5. **Кастомизируйте** — создавайте агентов под свои задачи
